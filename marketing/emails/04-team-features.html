Built for teams, not just you.

You've been running models on Cloudach for a week. If you're building something others depend on — or about to share your setup — here's what you should know about team features.

🔑

Per-team API keys with rate limits

Issue separate API keys for different teams, services, or environments. Set per-key rate limits and usage quotas so one service can't starve another.

📊

Usage dashboard & cost attribution

See token consumption broken down by API key, model, and time period. Export as CSV for billing reconciliation or chargeback to internal teams.

🔁

Autoscaling with zero config

Define min/max replicas and let Cloudach scale based on request queue depth. Scale to zero when idle — only pay for active inference time.

📋

Request logs & audit trail

Every request is logged with latency, token count, and key identifier. Tail logs in real-time from the dashboard or ship them to Datadog/CloudWatch.

🔒

Private VPC deployment (Pro+)

Run inference inside your own VPC with no public egress. Meets HIPAA, SOC 2, and GDPR requirements for regulated industries.
